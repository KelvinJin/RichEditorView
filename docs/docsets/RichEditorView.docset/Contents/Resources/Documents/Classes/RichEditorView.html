RichEditorView

public class RichEditorView: UIView

RichEditorView is a UIView that displays richly styled text, and allows it to be edited in a WYSIWYG fashion. It is a thin wrapper around a UIWebView utilizing the contentEditable APIs available in JavaScript.

It can be used just like any other UIView, either in code or in a storyboard or xib.

// In `viewDidLoad` of a `UIViewController`
let editor = RichEditorView(frame: CGRect(x: 0, y: 0, width: 320, height: 200))
view.addSubview(editor)

After that, when the user is inputting text, you will probably want to allow them to customize the text. This is done by calling the methods in the Rich Text Editing section below.

You will probably want a toolbar in order to assist the user in selecting text editing options. You can use the RichEditorToolbar included in the library, or create your own custom view, setting it to the inputAccessoryView.

Check out the sample project for a more complete example of how it can be used.

UIWebViewDelegate
  • webView(_:shouldStartLoadWithRequest:navigationType:)

    Delegate method for UIWebView. Is called when a URL will be loaded, and returns whether or not the UIWebView should continue with loading the request.

    This is also the main hub for communication between the JavaScript context inside the UIWebView to the RichEditorView. Inside the JavaScript, commands are queued, and when a URL with the scheme re-callback:// is loaded, the RichEditorView fetches the commands, and performs them sequentially.

    More detail about the performed actions, or the ability to add custom actions, can be found below in performCommand.

    Declaration

    Swift

    public func webView(webView: UIWebView, shouldStartLoadWithRequest request: NSURLRequest, navigationType: UIWebViewNavigationType) -> Bool

    Parameters

    webView

    Internal UIWebView that is being displayed.
    request

    NSURLRequest that is about to be loaded
    navigationType

    UIWebViewNavigationType of what type of navigation will be performed.

    Return Value

    Whether or not the UIWebView should continue loading the request. For the case of RichEditorView or custom events, it returns false, as the actions are taken care of already.

Utilities
  • runJS(_:)

    Runs some JavaScript on the UIWebView and returns the result. If there is no result, returns an empty string.

    JavaScript can be written just as if it were being written in the Safari Debug Console. For example:

    editor.runJS("3 + 5")               // returns "8"
editor.runJS("RE.editor.innerHTML") // returns the inner HTML of the editor
editor.runJS("nonsense strings")    // returns the empty string because it is nonsense

    Swift

    public func runJS(js: String) -> String
